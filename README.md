# Company-Policy-assistant

The Secure Role-Aware Enterprise Policy Intelligence Assistant (SRE-PIA) is a production-ready, full-stack application designed to modernize internal corporate governance by replacing traditional keyword-based document searching with context-aware natural language processing. Built with a decoupled architecture, the system employs React and TypeScript for a type-safe, fluid frontend UI, and FastAPI as the high-performance asynchronous backend api layer. User identity management, route protection, and credential hashing are strictly implemented via Passlib, Bcrypt, and JSON Web Tokens (JWT). Corporate documents are dynamically parsed, split into contextual blocks, converted into high-dimensional embeddings, and natively persisted alongside transactional application tables in a PostgreSQL database using the SQLAlchemy ORM, Alembic migration engine, and the pgvector extension.

To elevate this project beyond generic industry baselines, the architecture introduces advanced Vector-Level Role-Based Access Control (RBAC) and precise document lineage tracking. When an employee queries the system (e.g., "How many casual leaves are allowed?"), the backend interceptor automatically maps the user’s JWT authorization claims to the database query layer, ensuring semantic retrieval is restricted solely to policy documents the user is permitted to view. The system then merges the verified text chunks with the user's prompt to generate a hallucination-free, highly accurate response. The user interface completes the cycle by rendering verifiable, interactive source citation cards indicating the exact document title and page number, bridging the gap between conversational AI and enterprise data security.
